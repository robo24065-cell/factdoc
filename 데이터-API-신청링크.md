# 공공데이터 API — 직접 신청 링크 (이름 검색 X, 바로 이 링크로)

> data.go.kr에서 이름으로 검색하면 비슷한 게 수백 개 떠서 헷갈림. **아래는 우리 제품이 실제로 쓰는 정확한 데이터셋 상세페이지 직링크**다. 각 페이지에서 **[활용신청]** 버튼을 누르면 인증키(serviceKey)가 발급된다.
> 검증일: 2026-06-15 (제목·제공기관 직접 확인함).

## 1. 질병관리청 (주최기관 — 필수)

### ① 전수신고 감염병 발생현황  ★현재 사용 중 (대시보드 실시간 감염병 패널)
- 제목: **질병관리청_전수신고 감염병 발생현황**
- 직링크: https://www.data.go.kr/data/15139178/openapi.do
- 용도: 유행/급증 감염병 Top N·추세 (대시보드 ⑤, §13.10a). 라이선스 상대적으로 열림.

### ② 국가건강정보포털 건강정보 콘텐츠 API  ★코퍼스 확장 블로커 — data.go.kr 아님!
- **data.go.kr에 없다.** 질병청 포털에서 직접 신청한다(그래서 검색해도 안 나옴).
- OpenAPI 신청: https://health.kdca.go.kr/healthinfo/biz/health/portalUseGuidance/openApiReqst/openApiReqstRegist.do
- 활용방법 안내: https://health.kdca.go.kr/healthinfo/biz/health/portalUseGuidance/hlthinsReqst/hlthinsReqstMth.do
- 문의: 02-2030-6602 (평일 09–17시, 12–13시 제외)
- 용도: 질병별 원인·증상·치료·예방 **국가 공식 의학정보** = 코퍼스/그라운딩의 1차 인덱스. **이걸 받아야 8개 질환 → 수백 질환으로 확장 가능.**
- ⚠ 라이선스: 공공누리 **제4유형**(출처표시+비상업+변경금지). 원문 통째 재배포 금지 → 출처 포인터+짧은 인용+자체 요약만(§13.8 준수).

## 2. 식품의약품안전처 (보조 — 건기식 룰·제품 조회)

### ③ 의약품개요정보(e약은요)  ← "박카스/타이레놀 먹으면" 제품 자동조회용
- 제목: **식품의약품안전처_의약품개요정보(e약은요)**
- 직링크: https://www.data.go.kr/data/15075057/openapi.do
- 용도: 일반의약품 효능·용법·주의·상호작용·부작용. 제품/성분 분석 카드 자동화.

### ④ 건강기능식품 개별인정형 정보  ← 건기식 룰테이블
- 제목: **식품의약품안전처_건강기능식품 개별인정형 정보**
- 직링크: https://www.data.go.kr/data/15074311/openapi.do
- 용도: 인정 원료·일일섭취량·인정 기능성 → "이 건기식이 ○○병 치료" 자동 허위 룰(§13.2·§6).

### ⑤ 건강기능식품 기능성 원료인정 현황
- 제목: **식품의약품안전처_건강기능식품 기능성 원료인정 현황**
- 직링크: https://www.data.go.kr/data/15058359/openapi.do
- 용도: 원료↔인정 기능성 매핑 룰테이블 채우기.

## 활용신청 후 할 일
1. 각 페이지 **[활용신청]** → 사유 입력 → 인증키(일반/디코딩 serviceKey) 발급(보통 즉시~1일).
2. 발급 키를 레포에 커밋 금지 → `.env`(`VITE_*` 아님, 서버/배치용) 또는 GitHub Actions Secrets.
3. **per-request 외부호출 금지**(§4·§13.7) → GitHub Actions cron / pg_cron 배치로 주기 수집해 Supabase에 캐시, 프론트는 캐시만 읽음.
4. data.go.kr 키는 **승인까지 Forbidden**일 수 있음(현재 1471000 키 상태). 활용신청 승인 후 호출 가능.

## 이미 쓰는/탈락
- ✅ 사용 중: ① 감염병 발생현황 (대시보드 패널·클릭 퍼널).
- 🔜 키 발급 시 즉시 연동: ②(코퍼스 대량적재) · ③④⑤(제품/건기식 자동화).
- 병무청 개인 병역판정 기록 = API 미제공(데이터안심구역 내 분석만, §3) → 의존 안 함.
