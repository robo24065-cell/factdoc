## 결정 로그 (이유 포함)
- 부문 = **제품·서비스 개발** (작동 시제품 필수, 대상 350만원).
- 코어 데이터 = **질병청** (식약처가 주가 되지 않도록; 주최기관 데이터 미사용 시 0점 회피).
- 식약처 = **보조 모듈**(건강기능식품 주장 한정). 빼도 제품 성립하게 설계.
- DB = **Supabase(Postgres+pgvector)** — Firebase NoSQL은 클레임 그래프/관계형·벡터에 부적합.
- 독창성 = 데이터 구조화·연결 + 자체 판정엔진. 플러밍은 기성품 조립.
- 빌드 순서 = **수직 슬라이스 우선**, 항상 데모 가능 상태 유지.
- **(06-14) 폭+깊이 둘 다 간다**: 손라벨 대신 자동 추출 파이프라인 + 검증 티어로 폭 확장, 초집중은 4개 모트에. (사용자 결정: 마감을 제약으로 보지 않음. 단 7/7은 외부 게이트라 항상 작동 슬라이스 유지.)
- **(06-14) 근거수준은 LLM이 아니라 출처가 결정** (결정론 룰) — "누가 매겼나" 심사 누수 차단.
- **(06-14) 판정 위치 = Supabase Edge Functions**, Cloudflare Workers 금지(무료 CPU 10ms).
- **(06-14) 임베딩 = BGE-m3 1024d 로컬** (pgvector 2000차원 한계 회피·쿼터 0).
- **(06-14) "벌크 로컬 소유 코퍼스" 폐기** → 포인터+인용+요약+메타데이터. 질병청 4유형(비상업·변경금지) 대응 + 제출 전 043-719-7095 확인.

- **(06-14) 추가 기능은 5개만**(Why-Trace·유병률 정합성·Span Grounding·근거수준 자동등급·커버리지 라우터). 채널확장·식단추천·인바디·출력재포장은 로드맵 강등(초점·공공데이터 감점 회피).
- **(06-14) 대시보드는 PC/태블릿/모바일 3종 레이아웃 별도** + 마이페이지(KNHANES 맞춤맥락) + 비만 카테고리 추가.
