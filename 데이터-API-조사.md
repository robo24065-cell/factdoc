# FactDoc 활용 공공데이터 API 조사 (2026-06-17)

> 조달청 18개 승인 API + 추가 필요 API의 **제공 데이터·필드·FactDoc 활용처**. 병렬 웹조사 자동생성(10에이전트) → 정제본.
> ⚠ **정확한 데이터셋ID는 확인됨**(아래). 단 **개별 응답 영문 필드명·오퍼레이션 풀목록은 활용신청 후 Swagger 명세/응답메시지로 최종 확인** 필요(포털 Swagger가 JS 렌더라 웹조사로 일부 미확정). 필드명을 추측·날조하지 않음.

## 0. 핵심 결론 (FactDoc 관점)
- **조달청 데이터의 FactDoc 역할 = "방역물자 조달 동향 → 인포데믹/유행 선행지표"**. 마스크·진단키트·해열제·백신부자재 등 방역물자의 **발주계획→사전규격→입찰공고→낙찰→계약** 흐름이 정부의 방역 대응을 시간순으로 드러냄 → **프리벙킹(선제검증) 레이더의 외부 선행 트리거**(§아이디어1).
- **단, 판정 근거 아님**: 조달 데이터는 발주기관 행정행위일 뿐 감염병 발생의 직접 의학근거가 아님 → **"조기경보 트리거"로만**, 4단계 판정의 근거 체인에는 넣지 않는다(질병청·식약처 코어 유지).
- **대회 점수**: 조달청은 주최기관(병무청·방위사업청·질병청)이 아니므로 "주최기관 데이터 필수"엔 카운트 안 됨 → **공공데이터 활용 폭(독창적 융합)·발전가능성** 어필용. 주최기관 요건은 질병청 코어가 이미 충족.
- 공통 사양: 거의 전부 **REST · JSON+XML · 실시간 갱신 · 무료 · 개발계정 1,000회/일**(운영계정 증액).

## 6. ★추가 필요 API (FactDoc 신규 기능용 — 활용신청 권장)
### 6-1. 인구밀집도(확산률 시각화용) — 사용자 요청
- **(권장) 통계청 SGIS 총조사 주요지표 API** — 시도 단위 `ppltn_dnsty`(인구밀도 명/㎢) **직접 제공** + `tot_ppltn`·`avg_age`·`tot_family`. 인증키 필요, 연1회. → https://sgis.kostat.go.kr/developer/html/newOpenApi/api/dataApi/census.html (국가데이터처 `mods.go.kr` 이전 중)
- 행안부 주민등록 인구·세대현황 OpenAPI: 15108071(법정동)/15108065(행정동) — 시도 인구 합산용(밀도 미제공).
- e-나라지표 "지역별 인구 및 인구밀도": https://www.index.go.kr/unity/potal/main/EachDtlPageDetail.do?idx_cd=1007 (시도별 면적·인구밀도 표)
- **구현 메모**: 17개 시도 면적은 공개 고정값이라 **하드코딩 가능** → 기존 EID 인구추정(`sidoPop`)÷면적으로 밀도 산출하면 **신규 API 없이도 1차 구현 가능**. SGIS는 정밀화 옵션.

### 6-2. 검색어 트렌드(프리벙킹 선행지표) — 민간 API
- **네이버 데이터랩 통합검색어 트렌드** (민간, 네이버 개발자센터): `POST https://openapi.naver.com/v1/datalab/search`
  - 요청: `startDate`·`endDate`·`timeUnit`(date/week/month)·`keywordGroups`(최대5)·`keywords`(그룹당20)·`device`·`ages`·`gender`
  - 응답: `results[].data[].period`·`ratio`(⚠ **상대지수**, 절대건수 아님). 일/주/월. 무료(등록 후).
  - **활용**: "당뇨 완치"·"○○즙 효능"·신종 감염병명 검색량 급증(ratio 스파이크)을 가짜뉴스 확산 조기경보로 → 검증 우선순위 큐 정렬. **주최기관 데이터 아님 → 점수 무관, 보조.**
- 구글 트렌드 API: 알파 비공개(신청제) — 현재 일반 발급 불가.

### 6-3. 식약처 부당광고(자동 반증근거) — §13.2 `반증·규제근거`
- **식품의약품안전처_식품 허위·과대광고정보** 데이터셋ID **15058599** → http://apis.data.go.kr/1471000/FoodFlshdErtsInfoService03/getFoodFlshdErtsList_2
  - 확인 항목: **위반법령·행정처분일자·위반내용** (제품명/업체명 등 영문키 확인필요). JSON/XML.
  - 원천: 식품안전나라 "식품 등 부당한 표시·광고" 게시판(엑셀 다운로드).
  - **활용**: "식품 X가 ○○병 치료/예방"류 주장과 트리플 매칭 시, **국가가 실제 적발·처분한 동일 문구를 자동 반증근거로 발화** → 식약처 룰을 "법규상 불가"에서 "실제 처분 사례 존재"로 강화 + 처분일 기반 캐시 무효화 트리거.

## 7. 우선순위 권고
1. **식약처 부당광고(15058599)** — 코어 판정 강화(반증근거), 즉시 가치. **1순위.**
2. **인구밀도(SGIS/하드코딩)** — 확산률 시각화. 신규 API 없이 1차 가능, SGIS로 정밀화.
3. **조달 발주계획(15129462)+입찰공고(15129394) 물품구분 방역키워드** — 프리벙킹 레이더 선행지표. 배치 캐시(GitHub Actions)로 수집, per-request 호출 금지(§13.7).
4. 네이버 데이터랩 — 프리벙킹 보조(민간, 점수 무관).
5. 낙찰/계약/통계/누리장터 — 발전가능성 슬라이드용 보조(필요 시).
