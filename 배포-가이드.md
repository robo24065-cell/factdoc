# FactDoc 배포 가이드 — "내가(사용자가) 직접 해야 하는 것"

> 코드는 전부 준비돼 있습니다. 아래는 **제가(AI) 대신 할 수 없어 사용자가 직접 실행해야 하는** 작업입니다.
> 안전 정책상 비밀키 등록·외부 배포는 사용자 손으로 해야 해서, 명령을 그대로 복사해 터미널에 붙여넣으면 됩니다.
> 키·비밀번호는 모두 `api.txt`(레포에 커밋 안 됨)에 있습니다. 최종 갱신: 2026-06-17.

## 0. 준비 (한 번만)
1. **Node.js** 설치돼 있으면 됩니다(이미 개발 중이니 OK).
2. 터미널(PowerShell)을 열고 프로젝트 폴더로 이동:
   ```powershell
   cd C:\Users\PC\Downloads\공모전
   ```
3. Supabase CLI는 설치 불필요 — `npx supabase ...`로 매번 받아 씁니다.

## A. Supabase Edge Functions 배포 + 비밀키 등록 (엔진 풀지능 켜기)
> **왜?** 지금은 Gemini(LLM)가 안 붙어서, 지저분한 문장 파싱·능동 설명이 '로컬 폴백'으로만 돕니다.
> 아래를 하면 **"엠폭스 접촉으로 옮나요?" 같은 자연어를 똑똑하게 파싱·설명**하는 풀지능이 켜집니다.
> (판정 자체는 LLM이 안 하고 룰·그래프가 합니다 — LLM은 파싱/설명만. §6②)

### A-1. Supabase 로그인 (브라우저 1회 인증)
```powershell
npx supabase login
```
- 브라우저가 열리면 로그인 → 토큰 자동 저장.
- (또는 토큰 직접 입력: `$env:SUPABASE_ACCESS_TOKEN="<api.txt의 SUPABASE_ACCESS_TOKEN 값>"` 후 다음 단계)

### A-2. Gemini 키 풀을 Supabase 비밀키로 등록 (1줄)
> 🔒 **실제 키 값은 `api.txt`의 `GEMINI_API_KEYS=` 줄에 있습니다**(레포에 커밋 안 됨). 아래 `<...>` 자리에 그 값을 그대로 붙여넣으세요.
```powershell
npx supabase secrets set GEMINI_API_KEYS="<api.txt의 GEMINI_API_KEYS 콤마구분 값 4개>" --project-ref fcezfdnymianzxrezkdf
```
- 키 4개를 콤마로 등록 → 한 키가 한도 초과(429)면 자동으로 다음 키·다음 모델로 폴백.

### A-3. Edge Functions 5개 배포 (1줄)
```powershell
npx supabase functions deploy parse-claim explain-info explain-verdict embed-text review-answer --project-ref fcezfdnymianzxrezkdf --no-verify-jwt
```
- `parse-claim`(주장→트리플 파싱), `explain-info`/`explain-verdict`(쉬운 설명), `embed-text`(시맨틱 캐시 임베딩), `review-answer`(불만족 답변 AI 1차검토).

### A-4. 확인
- 사이트(홈)에서 "엠폭스는 어떻게 옮나요?" 입력 → 능동적인 설명이 나오면 성공.
- 안 되면: `npx supabase functions list --project-ref fcezfdnymianzxrezkdf`로 배포 여부, Supabase 대시보드 > Edge Functions > Logs로 오류 확인.

## B. Supabase DB 마이그레이션 적용 (검토 큐·캐시·피드백 테이블)
> **왜?** 검토 큐·시맨틱 캐시·부실응답 피드백을 **여러 기기에서 공유**하려면 DB 테이블이 필요합니다.
> (안 해도 로컬 모드로는 동작 — 단일 기기 localStorage. 발표/운영하려면 적용 권장.)

### 방법 ① CLI 한 번에 (권장)
```powershell
npx supabase link --project-ref fcezfdnymianzxrezkdf
npx supabase db push
```
- `link` 시 DB 비밀번호 물어보면 `api.txt`의 `SUPABASE_DB_PASSWORD` 값 입력.
- `db push`가 `supabase/migrations/0001~0010.sql`을 순서대로 적용.

### 방법 ② 대시보드에 붙여넣기 (CLI 안 되면)
- Supabase 대시보드 > **SQL Editor** > `supabase/migrations/` 의 `0001`부터 `0010`까지 **순서대로** 내용 복사→실행.

### 확인
- 관리자(`/admin`) > 대시보드 상단이 "● DB 연결됨"으로 바뀌면 성공.

## C. 신체 표준 통계 자동갱신 키 (선택 — 마이페이지 또래 비교 자동 최신화)
> **왜?** 병무청·연령별 신체 통계를 매년 사람 손 없이 자동 갱신하려면 KOSIS 키가 필요합니다.
> (안 해도 현재 검증된 시드 데이터로 동작 — 단, 자동 최신화는 안 됨.)

1. **KOSIS 키 발급(무료)**: https://kosis.kr/openapi/ → 로그인 → "활용신청" → 인증키 발급.
2. **GitHub 레포 시크릿 등록**: 레포 > Settings > Secrets and variables > Actions
   - `New repository secret` → 이름 `KOSIS_KEY`, 값 = 발급받은 키.
   - (선택) `Variables` 탭 > `BODY_KOSIS_MMA_TBL` = 병무청 병역판정 평균 통계표 tblId(KOSIS orgId=144에서 확인).
3. 끝. 매월 1일 `update-bodyspec.yml`이 자동 실행 → `bodyspec.ts` 최신화.

## D. 프론트엔드 환경변수 (Cloudflare Pages — 이미 연동돼 있으면 불필요)
빌드 환경변수에 다음만(공개 키라 프론트 노출 OK — 값은 `api.txt` 참조):
```
VITE_SUPABASE_URL=<api.txt의 SUPABASE_URL>
VITE_SUPABASE_PUBLISHABLE_KEY=<api.txt의 SUPABASE_PUBLISHABLE_KEY>
```
> ⚠ `SUPABASE_SECRET_KEY`·`DB_PASSWORD`·`GEMINI_*`·`ACCESS_TOKEN`은 **프론트 .env에 절대 넣지 말 것**(Edge Function/GitHub Secret 전용).
